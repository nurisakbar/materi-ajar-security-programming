4.1 Securing Linux

Melindungi sistem dari ancaman internal dan eksternal

Prinsip Keamanan Linux

  • Principle of Least Privilege: User hanya memiliki akses yang diperlukan
  • Defense in Depth: Multiple layers of security
  • Regular Updates: Patch security vulnerabilities
  • Monitoring and Logging: Monitor system activities

Area Keamanan Linux

  • User Management: Strong passwords, account policies
  • File Permissions: Proper ownership, chmod
  • Network Security: Firewall, disable unnecessary services
  • System Hardening: Disable root login, fail2ban
  • Logging: syslog, auditd, log rotation

SSH Hardening

# /etc/ssh/sshd_config
Protocol 2
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
MaxAuthTries 3
ClientAliveInterval 300

Firewall Configuration

# UFW (Ubuntu)
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable

4.2 Securing Database

Melindungi database dari akses tidak sah dan serangan

Ancaman Database

  • SQL Injection: Sudah dibahas di Day 2
  • Unauthorized Access: Weak passwords, default credentials
  • Privilege Escalation: Excessive user privileges
  • Data Exposure: Unencrypted data, unauthorized backups

Prinsip Keamanan Database

  • Least Privilege: User hanya memiliki privilege yang diperlukan
  • Encryption: Data at rest dan in transit
  • Access Control: Strong authentication, network restrictions
  • Auditing: Log semua aktivitas penting
  • Regular Updates: Patch security vulnerabilities

MySQL User Management

-- Create application user
CREATE USER 'app_user'@'localhost' 
IDENTIFIED BY 'StrongPassword123!';

-- Grant minimal privileges
GRANT SELECT, INSERT, UPDATE, DELETE 
ON security_training.* 
TO 'app_user'@'localhost';

-- Remove anonymous users
DELETE FROM mysql.user WHERE User='';

4.3 Using Encryption

Mengubah data menjadi format yang tidak dapat dibaca tanpa kunci

Jenis-jenis Encryption

Symmetric Encryption

Satu kunci untuk encrypt dan decrypt

Contoh: AES, DES

Cepat dan efisien

Asymmetric Encryption

Dua kunci: public dan private

Contoh: RSA, ECC

Lebih aman tapi lebih lambat

Hashing

One-way function (tidak dapat di-decrypt)

Contoh: SHA-256, bcrypt

Use Cases Encryption

  • Data at Rest: Encrypt files di disk, database, backups
  • Data in Transit: HTTPS/TLS, VPN, encrypted email
  • Password Storage: Hash passwords (bcrypt, argon2)

Encryption Best Practices

  • Gunakan strong algorithms (AES-256, RSA-2048+)
  • Manage keys securely
  • Rotate encryption keys regularly
  • Use proper key derivation functions
  • Don't store keys with encrypted data

4.4 Securing Network Connection: SSL and SSH

SSL/TLS: Mengamankan komunikasi melalui jaringan

SSH: Protokol untuk akses remote yang aman

SSL/TLS

  • Encryption: Mengenkripsi data yang ditransmisikan
  • Authentication: Memverifikasi identitas server
  • Integrity: Memastikan data tidak diubah

SSL/TLS Certificates

Self-Signed

Untuk development/testing

Browser akan menampilkan warning

CA-Signed

Dikeluarkan oleh Certificate Authority

Dipercaya oleh browser

Contoh: Let's Encrypt (gratis)

SSH Security

  • Key-based authentication (lebih aman dari password)
  • Disable root login
  • Change default port (optional)
  • Use strong ciphers
  • Implement fail2ban

4.5 Securing Web Server

Melindungi web server dari berbagai ancaman

Ancaman Web Server

  • DDoS Attacks: Overwhelm server dengan traffic
  • Directory Traversal: Akses file di luar web root
  • Server Misconfiguration: Default settings yang tidak aman
  • Information Disclosure: Error messages yang terlalu informatif
  • Outdated Software: Known vulnerabilities

Web Server Hardening

  • Remove unnecessary modules
  • Disable directory listing
  • Set proper file permissions
  • Configure security headers
  • Enable logging
  • Rate limiting
  • WAF (Web Application Firewall)

Security Headers

X-XSS-Protection: 1; mode=block
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
Strict-Transport-Security: max-age=63072000
Content-Security-Policy: default-src 'self'

Sesi Praktik Day 4

Praktik securing environment yang dapat diikuti peserta

Praktik 1: Hardening Linux Server

Tujuan: Mengamankan server Linux dengan berbagai teknik hardening

Langkah-langkah:

  1. Update sistem: sudo apt update && sudo apt upgrade
  2. Disable root login SSH
  3. Setup firewall dengan UFW
  4. Install dan konfigurasi fail2ban
  5. Setup automatic security updates
# 1. Update system
sudo apt update
sudo apt upgrade -y

# 2. Disable root login
sudo nano /etc/ssh/sshd_config
# Set: PermitRootLogin no
sudo systemctl restart sshd

# 3. Setup UFW firewall
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow ssh
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 4. Install fail2ban
sudo apt install fail2ban -y
sudo systemctl enable fail2ban
sudo systemctl start fail2ban

# 5. Setup automatic updates
sudo apt install unattended-upgrades -y
sudo dpkg-reconfigure -plow unattended-upgrades

Praktik 2: Mengamankan Database MySQL

Tujuan: Mengimplementasikan security best practices untuk MySQL

Langkah-langkah:

  1. Jalankan mysql_secure_installation
  2. Buat user dengan least privilege
  3. Enable SSL untuk koneksi database
  4. Setup database backup
  5. Enable audit logging
-- 1. Secure installation
sudo mysql_secure_installation

-- 2. Create application user with minimal privileges
CREATE USER 'app_user'@'localhost' IDENTIFIED BY 'StrongPassword123!';
GRANT SELECT, INSERT, UPDATE, DELETE ON myapp.* TO 'app_user'@'localhost';
FLUSH PRIVILEGES;

-- 3. Remove anonymous users
DELETE FROM mysql.user WHERE User='';
FLUSH PRIVILEGES;

-- 4. Enable SSL (generate certificates first)
ALTER USER 'app_user'@'localhost' REQUIRE SSL;

-- 5. Enable general query log
SET GLOBAL general_log = 'ON';
SET GLOBAL log_output = 'FILE';

Praktik 3: Setup SSL/TLS dengan Let's Encrypt

Tujuan: Mengimplementasikan HTTPS dengan sertifikat SSL/TLS gratis

Langkah-langkah:

  1. Install Certbot
  2. Generate SSL certificate
  3. Konfigurasi Apache/Nginx untuk HTTPS
  4. Setup auto-renewal
  5. Test SSL configuration
# 1. Install Certbot
sudo apt install certbot python3-certbot-apache -y

# 2. Generate SSL certificate
sudo certbot --apache -d example.com -d www.example.com

# 3. Test auto-renewal
sudo certbot renew --dry-run

# 4. Setup cron for auto-renewal
sudo crontab -e
# Add: 0 0 * * * certbot renew --quiet

# 5. Test SSL configuration
# Visit: https://www.ssllabs.com/ssltest/

# Apache configuration (auto-generated by certbot)
# /etc/apache2/sites-available/example.com-le-ssl.conf
<VirtualHost *:443>
    ServerName example.com
    SSLEngine on
    SSLCertificateFile /etc/letsencrypt/live/example.com/fullchain.pem
    SSLCertificateKeyFile /etc/letsencrypt/live/example.com/privkey.pem
</VirtualHost>

Praktik 4: Konfigurasi Security Headers

Tujuan: Mengimplementasikan security headers untuk web server

Langkah-langkah:

  1. Buat file konfigurasi security headers
  2. Implementasikan di Apache/Nginx
  3. Test dengan securityheaders.com
  4. Implementasikan CSP (Content Security Policy)
# Apache: /etc/apache2/conf-available/security-headers.conf
<IfModule mod_headers.c>
    # XSS Protection
    Header set X-XSS-Protection "1; mode=block"
    
    # Prevent MIME type sniffing
    Header set X-Content-Type-Options "nosniff"
    
    # Prevent clickjacking
    Header set X-Frame-Options "DENY"
    
    # HSTS (HTTP Strict Transport Security)
    Header always set Strict-Transport-Security "max-age=63072000; includeSubDomains; preload"
    
    # Content Security Policy
    Header set Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'; style-src 'self' 'unsafe-inline';"
    
    # Referrer Policy
    Header set Referrer-Policy "strict-origin-when-cross-origin"
</IfModule>

# Enable the configuration
sudo a2enconf security-headers
sudo systemctl reload apache2

Praktik 5: Implementasi File Encryption

Tujuan: Mengenkripsi file sensitif menggunakan OpenSSL

Langkah-langkah:

  1. Generate encryption key
  2. Buat script untuk encrypt file
  3. Buat script untuk decrypt file
  4. Implementasikan di aplikasi PHP
# 1. Generate encryption key
openssl rand -base64 32 > encryption_key.txt
chmod 600 encryption_key.txt

# 2. Encrypt file
openssl enc -aes-256-cbc -salt -in sensitive_file.txt -out sensitive_file.txt.enc -pass file:encryption_key.txt

# 3. Decrypt file
openssl enc -aes-256-cbc -d -in sensitive_file.txt.enc -out sensitive_file_decrypted.txt -pass file:encryption_key.txt

// PHP implementation
function encryptFile($filepath, $key) {
    $data = file_get_contents($filepath);
    $iv = openssl_random_pseudo_bytes(16);
    $encrypted = openssl_encrypt($data, 'AES-256-CBC', $key, 0, $iv);
    return base64_encode($iv . $encrypted);
}

function decryptFile($encrypted_data, $key) {
    $data = base64_decode($encrypted_data);
    $iv = substr($data, 0, 16);
    $encrypted = substr($data, 16);
    return openssl_decrypt($encrypted, 'AES-256-CBC', $key, 0, $iv);
}

// Usage
$key = file_get_contents('encryption_key.txt');
$encrypted = encryptFile('sensitive.txt', $key);
file_put_contents('sensitive.txt.enc', $encrypted);

Praktik 6: Setup SSH Key Authentication

Tujuan: Mengganti password authentication dengan SSH keys

Langkah-langkah:

  1. Generate SSH key pair di client
  2. Copy public key ke server
  3. Disable password authentication
  4. Test koneksi SSH
# 1. Generate SSH key pair (on client)
ssh-keygen -t ed25519 -C "your_email@example.com"
# Save to: ~/.ssh/id_ed25519

# 2. Copy public key to server
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server_ip

# Or manually:
cat ~/.ssh/id_ed25519.pub | ssh user@server_ip "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

# 3. Secure the authorized_keys file
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# 4. Edit SSH config (on server)
sudo nano /etc/ssh/sshd_config
# Set: PasswordAuthentication no
# Set: PubkeyAuthentication yes
sudo systemctl restart sshd

# 5. Test connection
ssh -i ~/.ssh/id_ed25519 user@server_ip
